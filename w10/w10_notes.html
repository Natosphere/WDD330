Form Validation

Why?

  • Get the correct data in the correct format
  • Protect the data
  • Protect the server

Methods:

  • Built-in html form validation
    1. Required, minlength, maxlength, min/max, type, pattern
    2. css :valid/:invalid pseudo-class, :out-of-range
    3. Regex: pattern=”[Bb]anana|[Cc]herry”
  • JavaScript (constraint validation api)
    1. .validity
      • PatternMismatch, tooLong, maxLength, tooShort, rangeOverflow, rangeUnderflow, typeMismatch, valid, valueMissing
    2. Checkvalidity()
    3. setCustomValidity(‘Message’)

 

Fetch

  • credentials: ‘include’/’same-origin’/’omit’
  • Uploading json:
    1. method: ‘POST’/’PUT’
    2. headers: {Content-Type: ‘application/json’,},
    3. body: JSON.stringify(data),
  • Uploading file:
    1. fileField = document.querySelector('input[type="file"]');
    2. method: “PUT”
    3. body: formData
